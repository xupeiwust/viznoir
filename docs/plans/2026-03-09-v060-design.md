# viznoir v0.6.0 Design — Science Storyteller v2 + Compose Quality

## Scope

**Phase 1 (v0.6.0):** A + B 병렬 구현, 품질 100% 검증
**Phase 2 (v0.7.0):** C. Auto-Story Pipeline (inspect_physics → cinematic_render × N → compose_assets 원스텝)

### A. Science Storyteller v2
- L2 FieldTopology: vortex detection (Q-criterion), critical points, centerline probes, gradient analysis
- L3 CaseContext: boundary conditions, transport properties, solver info, mesh quality, derived quantities (Re, Ma, Pr)
- `inspect_physics` MCP tool: L2 + L3 통합, `analyze_data` deprecate
- OpenFOAM parser first, ContextParser Protocol로 확장 가능

### B. compose_assets Quality Improvement
- Layout engine 파라미터화 (cols, responsive padding, min panel width)
- Font system (플랫폼별 탐색, CJK 지원, 자동 스케일링)
- Label visibility (반투명 배경 박스, truncation, collision avoidance)
- Grid sparse 해결 (빈 셀 제거, auto-cols, aspect ratio 보존)

## Architecture

```
A: context/ (신규 모듈)
  context/models.py        ← L3 dataclass 6종
  context/parser.py        ← ContextParser Protocol + Registry
  context/generic.py       ← GenericContextParser (mesh quality only)
  context/openfoam.py      ← OpenFOAMContextParser (BC, solver, transport)

A: engine/topology.py (신규)
  ← L2: detect_vortices, detect_critical_points, extract_centerline_profiles,
         compute_gradient_stats, analyze_field_topology

A: tools/inspect_physics.py (신규)
  ← L2 + L3 통합 MCP tool

B: anim/compositor.py (수정)
  ← layout engine, font system, label visibility, grid sparse

B: tools/compose.py (수정)
  ← asset validation 강화
```

### Data Flow: inspect_physics

```
inspect_physics(file_path, case_dir?, fields?, probe_lines=3, vortex_threshold=0.01)
  → engine/readers.py: read_dataset(file_path)
  → engine/topology.py: analyze_field_topology(dataset, field) × N
  → context/parser.py: registry.get_parser(case_dir) → parser.parse(case_dir)
  → JSON: { field_topologies: FieldTopology[], case_context: CaseContext?, extraction_time_ms }
```

## Key Decisions

1. **viznoir = data extraction engine, LLM = physics interpreter** — 하드코딩 물리 지식 제거
2. **OpenFOAM parser first** — v0.6.0 범위. ContextParser Protocol로 CGNS, Exodus 등 확장 가능
3. **analyze_data deprecated, not removed** — 기존 사용자 호환, docstring warning
4. **compositor 개선은 backward-compatible** — 기존 compose_assets 호출 동작 유지
5. **File ownership separation** — agent 간 충돌 제로 보장
