## v0.7.0 — Autoexp: 자율 시각화 최적화 (2-3 weeks)

**핵심 아이디어**: Karpathy autoresearch 패턴을 시각화 파라미터 최적화에 적용.

### 설계 원리

```
autoresearch 3-파일 패턴 → viznoir 적용
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
prepare.py (불변)    →  engine/readers.py + evaluation harness
render.py  (수정대상) →  viz_params.py (카메라, 조명, 컬러맵, TF)
program.md (지시)    →  도메인별 목표 ("CFD 유동장의 최적 시각화")
metric     (단일)    →  ImageQualityMetric (복합 스코어)
```

### 새 모듈

| Module | Purpose |
|--------|---------|
| `engine/quality.py` | 렌더링 품질 메트릭 — edge entropy, contrast ratio, feature visibility, colormap utilization |
| `engine/presets.py` | 도메인별 검증된 파라미터 프리셋 라이브러리 (Git Ratcheting으로 축적) |
| `core/autoexp.py` | 자율 실험 루프 — modify → render → measure → keep/revert → log |
| `tools/auto_optimize.py` | `auto_optimize` MCP tool — 데이터셋에 최적 시각화 파라미터 자동 탐색 |

### 메트릭 설계

```python
@dataclass
class ImageQualityMetric:
    """Composite visualization quality score."""
    contrast: float      # Michelson contrast of rendered image
    edge_entropy: float  # Shannon entropy of edge-detected image (구조 가시성)
    field_coverage: float  # scalar range 활용률 (0~1)
    render_time_ms: float  # 렌더링 소요 시간

    @property
    def score(self) -> float:
        """Weighted composite — higher is better."""
        return (0.4 * self.contrast +
                0.3 * self.edge_entropy +
                0.3 * self.field_coverage)
```

### Git Ratcheting 플로우

```
1. 데이터셋 로드 (불변)
2. 현재 preset으로 렌더링 → baseline score
3. 파라미터 변경 (카메라 각도, 조명, 컬러맵, TF)
4. 렌더링 → new score
5. score 개선 → preset 저장, 악화 → revert
6. results.tsv에 모든 시도 기록
7. 반복 (budget 소진까지)
```

## v0.8.0 — Multi-Parser + 산업 데이터 검증 (3-4 weeks)

현재 한계: "VTK 예제 데이터셋으로만 검증됨"을 해소.

### 새 Context Parser

| Parser | 도메인 | 데이터 소스 |
|--------|--------|------------|
| `context/cgns.py` | 항공/터보 CFD | NASA TMR benchmark, DLR-F6 |
| `context/exodus.py` | 구조/열 FEA | CalculiX, Sierra/Solid |
| `context/dualsphysics.py` | SPH 유체역학 | DualSPHysics XML/VTK |
| `context/vtk_generic.py` | 범용 | VTI/VTS/VTP/VTU 메타데이터 |

### 산업 스케일 검증

| 데이터셋 | 크기 | 검증 항목 |
|----------|------|----------|
| DrivAerML | 8.8M cells, 629MB | 대용량 VTP 렌더링 성능, 메모리 |
| WindsorML | 388MB | VTU 대용량 reader 안정성 |
| OpenFOAM 15 cases | 170 VTU + 697 VTP | context parser 범용성 |
| CGNS NASA TMR | 218MB | 새 parser 정확도 |

### 성능 목표

- 100M cell 데이터셋에서 30초 이내 첫 렌더링
- GPU 메모리 12GB 이내 (RTX 4090 절반)
- 스트리밍 LOD: 원격 데이터셋 progressive 로딩

## v0.9.0 — Physics Guard: 환각 방지 (2-3 weeks)

현재 한계: "LLM 환각에 의한 물리적 왜곡 실시간 검증 불가"를 해소.

### 설계 원리 (3계층 구조의 Strategy 레이어 강화)

```
Strategy (메타프롬프트)  →  물리적 일관성 규칙 = Physics Guard
Process  (체이닝)       →  render → validate → accept/reject 루프
Action   (MCP/Skill)    →  기존 22개 tool + guard tool
```

### 새 모듈

| Module | Purpose |
|--------|---------|
| `guard/physics_rules.py` | 도메인별 물리 검증 규칙 (질량 보존, 에너지 보존, 양의 스칼라 등) |
| `guard/visual_rules.py` | 시각화 검증 규칙 (컬러맵 적절성, 스케일 일관성, 뷰포인트 의미) |
| `guard/validator.py` | 렌더링 후 자동 검증 — pass/warn/fail 분류 |
| `tools/validate_render.py` | `validate_render` MCP tool |

### 검증 규칙 예시

```python
# 물리 규칙
- "압력 필드에 diverging 컬러맵 사용 시 0점이 컬러맵 중앙이어야 함"
- "속도 크기는 음수가 될 수 없음 → sequential 컬러맵만 허용"
- "온도가 절대 0도 이하면 경고"
- "벡터 필드 streamline은 forward integration만 물리적 의미"

# 시각화 규칙
- "아이소서피스 값이 데이터 범위 밖이면 빈 결과 → 에러 대신 가이드"
- "카메라가 데이터 경계 밖에 있으면 빈 이미지 → 자동 리셋"
- "동일 렌더 2회 결과가 다르면 non-deterministic 경고"
```
